# Bandit

## Level 10

<pre>
    사용한 명령어
    - base64 (옵션 : d)

    배운 것

    base64 
    -> 이진(Binary) 데이터를 64개의 안전한 ASCII 문자로 인코딩하거나 디코딩할 때 사용
        옵션:
            -d
            -> decode(디코드)의 약자로 암호화된 문자를 디코딩 할때 사용

    *
    base64 -d data.txt 
    혹은
    base64 -decode data.txt 둘다 같은 의미로 기능

</pre>
## Level 11

<pre>
    사용한 명령어
    - tr

    배운 것

    tr [바꿀문자들] [바뀔문자들]
    -> 문자를 다른 문자로 1:1 치환하는 명령어

    *
        tr 'abc' 'xyz'라면
        a->x, b->y, c->z 이런식으로 치환됨
        여기서는 13번 뒤로 미뤄 A->N 이런식으로 됐으니까 다시 돌릴라면 13번 뒤로 미뤄야됨(26개가 있으니 순환)
        즉 tr 'A-Za-z' 'N-ZA-Mn-za-m' 이러면 
        대문자 A~Z, 소문자 a~z가 13번 뒤로 미뤄져서 N~M, n~m 구조를 이루게 됨
        즉 돌아온것임

        * 하지만 저러먼 오류뜨고 
        tr 'A-Za-z' 'N-ZA-Mn-za-m' 
        이런식으로 순서를 역으로 갈 순 없고 순서대로 해야됨 
        만약 Z까지 갔으면 A도 직접 설정해줘야됨 아니면 역주행하는거니까

        ** tr 'A-Z a-z' 'N-Z A-M n-z a-m' 이런식으로 띄어쓰기를 하면 공백도 문자열 취급해버리는 불상사가 생기기 때문에 공백 없이 해야됨

</pre>

## Level 12 ⭐️⭐️

<pre>
    사용한 명령어
    - mktemp -d
    - cp
    - xxd (옵션: -r)
    - >
    - rm
    - mv
    - gunzip
    - bunzip
    - tar
    - file

    - hexdump
    

    배운 것

    mktemp -d
    -> /tmp 안에 무작위의 이름을 가진 디렉토리 생성

    cp [원본] [목적지]
    -> 원본을 목적지 위치에 복사 (파일, 디렉토리 복사기능)

    xxd 
    -> 바이너리 파일을 hexdump(16진수) 형태로 출력
        옵션 :
            -r
            -> reverse의 약자로 원래 형식으로 돌려준다, 즉 hexdump를 원래 바이너리 파일로 복원
    
    명령어 [원본파일] > [결과를 저장할 파일] 
    -> 원본파일을 이용해서 명령어를 실행한 결과글 저 결과를 저장할 파일에 저장

    rm
    -> 파일 혹은 디렉토리를 지우는 명령어

    mv
    -> 파일이나 디렉토리의 위치를 옮겨주는 명령어,
        이름을 변경해주는 명령어

   *파일 압축해제 
    gzip [파일이름]
    -> 파일을 gzip 형태로 압축

    gunzip [파일이름]
    -> 파일 압축해제한 후 원본파일 삭제 
    gzip -d [파일이름]
    -> 파일 압축해제한 후 원본파일 삭제 

    tar -xvf [파일이름]
    -> 여러 파일을 하나의 tar 아카이브로 묶거나, tar 파일을 해제하는 명령어 (예, bin 형태의 파일해제)


    hexdump
    -> 파일이나 데이터를 16진수(Hex) 및 ASCII 값으로 출력


    *
        xxd
        -> ( 바이너리 → 16진수 ) 형태로 변경
        xxd -r
        -> ( 16진수 → 바이너리 ) 형태로 변경

        cp [복사할 파일] .
        -> 현재위치에 파일 복사, 만약 복사할 파일이 다른곳에 있으면 경로/복사할파일이름 이런식으로 입력

        명령어 [원본파일] > [결과를 저장할 파일] 
        -> 이때 [원본파일]과 [결과를 저장할 파일]이 같으면 안됨 왜냐하면 [결과를저장하는 파일]을 초기화 하고 원본파일을 넣는건데 
            아렇게 되면 원본파일을 초기화 하고 넣는거기 때문에 이미 초기화된 상태에서 넣을 내용이 없음

        mv [기존이름] [교환한 이름]
        -> 이런형식으로 이름, 형식 바꿀때 사용

        처음 압축을 해체한거를 저장해야하기때문에
        xxd -r data.txt > data 이렇게 사용해야함

        gunzip이든 bunzip2 명령어를 사용하든 해체할 파일의 형식이 그에 맞게 되어있어야됨
        예시로 gunzip을 하고싶으면 파일 형식이 파일이름.gz 이런식으로 되어있어야지 사용 가능

        이건 약간 노가다 
        file로 계속 형식확인하면서 그에 맞게 파일 형식 바꿔주고 압축 해제하면됨

</pre>

## Level 13

<pre>
    사용한 명령어
    - man
    - ssh (옵션 : i)
    - whoami

    배운 것
    
    man
    - man 뒤에 적은 명령어 설명을 보여줌

    ssh -i 열쇠파일 사용자@컴퓨터 포트번호
    -> 열쇠파일을 사용해서 입력한 컴퓨터에서 저 사용자로 들어간다
        옵션 :
            i
            -> Identity file(개인 키 파일)을 사용하라는 뜻

    whoami
    - 현재 사용자계정을 확인할수 있음

    * 
        ssh -i sshkey.private bandit14.localhost -p 2220
        -> 이미 나는 bandit 서버에 들어와있어서 다시 인터넷으로 갈 필요가 없기 때문에 localhost를 사용해서 
            개인키(i) sshkey.private bandit14@localhost 을 이용해서 bandit14로 사용자를 바꾸는것

            IP 주소 = 아파트 주소
            포트 번호 = 몇 호인지
                   or
            IP 주소 = 어떤 컴퓨터인지
            포트 번호 = 그 컴퓨터 안의 어떤 프로그램인지
        
        근데 밴딧에서는 bandit서버 안에서 ssh로 bandit서버로 들어가는게 정책상 안됨
        그래서 ssh키를 복사해서 맥북에 들고온 다음 맥북에서 bandit으로 접속해야됨
</pre>
