# Bandit

## Level 5

사용한 명령어
- man
- find
- cd
- cat

배운 것

find
-> 파일과 디렉터리를 조건에 맞게 검색하는 명령어

find . 
-> 현재 위치 아래에있는 파일과 폴더 전체 출력

find . -type f
-> 현재 위치 아래에 있는 파일만 출력

find . -size 1033c
-> 현재 위치 아래에 있는 파일 중 사이즈가 1033byte인 것만 출력 (c가 byte라는 의미)

*
man 명령어 치면 이 명령어에 대한 문서를 볼 수 있음
혹은 명령어 --help도 같은 기능

## Level 6

사용한 명령어
- ls
- cat
- cd
- find

배운것

find /
-> 루트(/)부터 시작해서 모든 파일과 디렉터리를 검색

find -user 유저이름
-> 유저 소유의 파일 찾기

find -group 그룹이름
-> 그룹 소유의 파일 찾기 

2>/dev/null
-> 에러출력 숨기기

*
소유자, 그룹을 다루기 때문에 .(현재위치가) /(루트)부터 찾아야됨

find / -user bandit7 -group bandit6 -size 33c
이걸 그대로 사용하면 권한이 없는 디렉토리가 많아서 Permission denied라는 오류가 뜨게됨
그리고 많이 떠서 제대로 못찾음

2>/dev/null 이걸 뒤에 붙여서 사용하면 Permission denied오류는 안보이고 깔끔하게 결과만 나옴
지금은 이게 에러메세지를 버리는 역할이라고만 생각

find / -user bandit7 -group bandit6 -size 33c 2>/dev/null 이런식으로

## Level 7

사용한 명령어
- ls
- cat
- grep


배운것

grep [옵션] '검색어' [파일이름]
-> 파일에서 검색어가 포함된 줄을 출력
    
    옵션
        -i: 대소문자 구분 없이 검색

다른 명령어 | grep '검색어'
-> 앞의 명령어에서 출력한 결과 중에 검색어가 포함된 줄을 출력

grep "검색어" 파일명
-> 파일에서 검색어가 포함된 줄을 출력

*
grep '검색어' 파일이름 → 파일을 직접 읽어서 검색
다른명령어 | grep '검색어' → 앞 명령어의 결과를 검색

|(파이프라인) 을 사용한 경우는
(예) ls | grep ".txt" 

    ls의 결과가   a.txt  라면
                b.txt
                hello.c
                main.cpp
    
    grep은 이 출력을 받아서 .txt의 형식만 출력해주는 형태임

즉 grep 앞의 명령어의 출력값을 받아서 검색하는것

그냥 grep "검색어" 파일명 이렇게 사용하면
파일을 열어서 검색하는거

## Level 8

사용한 명령어
- ls
- cat
- sort
- uniq

배운 것

sort 파일명 | uniq
-> 파일의 내용을 알파벳 순서대로 정렬한 뒤, 중복된 행을 제거, 고유한 값만 출력

sort 파일명 | uniq -d
-> 중복되어 여러번 나타난 행들만 출력

sort 파일명 | uniq -u
-> 중복되지 않고 하나만 존재하는 행만 출력

*
uniq는 같은 내용이 연속으로 있을 때만 중복을 제거하므로 보통 sort와 함께 사용한다.

uniq의 옵션이기 때문에 uniq 뒤에 옵션이 붙음
