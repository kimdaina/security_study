## Level 19

<pre>
    사용한 명령어
    - ls -l
    
    배운것

    setuid binary
    - 리눅스, 유닉스 시스템에서 프로그램이 실행하는 동안 해당파일의 소유자 권한을 일시적으로 부여하는 특수권한
</pre>

<pre>
    *  
        이 문제에서는 setUID가 설정되어있는 파일을 사용해야하기 때문에
        chmod를 사용하면안됨 
        chmod는 내가 소유한 파일의 권한을 바꾸는거지, 내 소유가 아닌데 권한을 바꿀 수는 없음

        ls -l이라는 명령어를 사용해서 파일 권한확인
        s가 들어간 권한이 있는 파일이여야됨 (setUID가 설정되어있는 파일)
        권한 [소유자] [그룹] [파일이름] 이런 형식으로 출력됨

        cat 파일을 실행하면 
        일반적으로 지금 명령어를 사용하는 사용자의 권한으로 실행됨

        홈 디렉토리에 있는 파일의 권한을 사용해야됨 

        whoami 
        → 현재 사용자 이름만 출력
        
        id 
        → 사용자, 그룹, UID, GID까지 모두 출력
        
        일반적으로 whoami를 치면 현재 명령어를 사용하고 있는 사용자 출력

        하지만

        * ./(홈 디렉토리에 있는 SetUID 실행 파일을 사용) whoami를 실행하면
            whoami가 저 파일의 권한으로 실행됨 (그러니까 저 파일의 프로그램을 실행 한뒤 whoami 실행)

        그러므로 그 소유자의 권한으로 파일을 실행하고 싶으면
        SetUID가 설정된 실행 파일을 먼저 실행하고 뒤에 명령어 실행
        (예) ./파일 cat /etc/password

        ** SetUID 실행 파일은 뒤에 전달한 명령어를 자신의 소유자 권한으로 실행한다.
</pre>
